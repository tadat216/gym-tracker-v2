# Foundation Setup Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold the gym-tracker-v2 monorepo so that `docker compose --profile dev up --build` boots a working backend + frontend + Postgres stack, with test infrastructure ready.

**Architecture:** Monorepo with `backend/` (FastAPI) and `frontend/` (React/Vite/TypeScript) at root. Docker Compose orchestrates services with `dev` and `prod` profiles. Two Postgres containers: one for dev/prod data, one for tests.

**Tech Stack:** FastAPI, SQLModel, asyncpg, Alembic, Vite, React, TypeScript, Tailwind CSS v4, shadcn/ui, Playwright, Vitest, Orval, Docker Compose

**Spec:** `docs/superpowers/specs/2026-03-18-foundation-setup-design.md`

**Files:**
- Create: `docker-compose.yml`
- Create: `docker/init-db.sh`

- [x] **Step 1: Create `docker/init-db.sh`**

```bash
#!/bin/bash
set -e

psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" --dbname "$POSTGRES_DB" <<-EOSQL
    SELECT 'CREATE DATABASE gym_tracker_dev'
    WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = 'gym_tracker_dev')\gexec
EOSQL
```

Must be executable (`chmod +x`). Mounted to `/docker-entrypoint-initdb.d/` on the postgres container.

- [x] **Step 2: Create `docker-compose.yml`**

Services:
- `postgres`: image `postgres:16-alpine`, port `5432:5432`, volume `pgdata`, env from root `.env`, mounts `docker/init-db.sh` to `/docker-entrypoint-initdb.d/`, healthcheck via `pg_isready`. No profile (always runs).
- `postgres-test`: image `postgres:16-alpine`, port `5433:5432`, volume `pgdata-test`, `POSTGRES_DB=gym_tracker_test`, profile `dev`. Healthcheck via `pg_isready`.
- `backend-dev`: build `backend/` target `dev`, port `8000:8000`, volume `./backend:/app`, `env_file: backend/.env`, depends on `postgres` (healthy), profile `dev`.
- `frontend-dev`: build `frontend/` target `dev`, port `5173:5173`, volume `./frontend:/app` + anonymous volume for `node_modules`, `env_file: frontend/.env`, profile `dev`.
- `backend-prod`: build `backend/` target `prod`, port `8000:8000`, `env_file: backend/.env`, depends on `postgres` (healthy), profile `prod`.
- `frontend-prod`: build `frontend/` target `prod`, port `80:80`, profile `prod`.

Volumes: `pgdata`, `pgdata-test`.

```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    DATABASE_URL: str
    TEST_DATABASE_URL: str
    SECRET_KEY: str
    ACCESS_TOKEN_EXPIRE_MINUTES: int = 10080
    CORS_ORIGINS: str

    model_config = {"env_file": ".env"}

    @property
    def cors_origins_list(self) -> list[str]:
        return [origin.strip() for origin in self.CORS_ORIGINS.split(",")]

settings = Settings()
```

```python
async def get_session():
```

## Task 9: API Generation Script

**Files:**
- Create: `scripts/generate-api.sh`

- [ ] **Step 1: Create `scripts/generate-api.sh`**

```bash
#!/bin/bash
set -e

BACKEND_URL="${BACKEND_URL:-http://localhost:8000}"
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
FRONTEND_DIR="$SCRIPT_DIR/../frontend"

echo "Fetching OpenAPI spec from $BACKEND_URL..."
curl -s "$BACKEND_URL/openapi.json" -o "$FRONTEND_DIR/openapi.json"

echo "Generating API client..."
cd "$FRONTEND_DIR"
npx orval

echo "Cleaning up..."
rm -f "$FRONTEND_DIR/openapi.json"

echo "Done! API client generated in frontend/src/api/"
```
